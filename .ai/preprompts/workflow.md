# Workflow

Always follow this order:

1. Understand the request
2. Read relevant files
3. Explain which files will change
4. Make a short plan
5. Implement the minimum viable version
6. Improve polish and edge cases
7. Review your own work
8. Summarize remaining work

Only implement one major feature at a time.

Never:
- Change too many files at once
- Mix unrelated features
- Skip explanation
- Skip review
