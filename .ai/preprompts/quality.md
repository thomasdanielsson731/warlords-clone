# Quality Expectations

Never stop at the first working implementation.

Every feature must include:
- Complete functionality
- Good visuals
- Clear UI feedback
- Edge case handling
- Manual test steps
- Smooth transitions
- Clear naming
- Good TypeScript types

Always ask:
- Is this good enough for a released game?
- Would this feel polished to a player?
- Does this still feel like Warlords II?

If not, continue improving.
